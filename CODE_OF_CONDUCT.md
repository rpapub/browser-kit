# Code of Conduct

## Purpose

This project exists to build useful software collaboratively.

The goal is a friendly environment where people with different levels of experience can contribute.


## Expected behavior

Please:

- be respectful
- assume good intentions
- provide constructive feedback
- explain reasoning
- help newcomers
- focus criticism on ideas, not people
- accept that others may have different technical opinions


## Unacceptable behavior

Examples include:

- personal attacks
- harassment
- discrimination
- hostile language
- deliberate intimidation
- repeated disruptive behavior
- dismissing contributors based on experience level


## Technical disagreement

Disagreement is expected.

When opinions differ:

- explain tradeoffs
- provide examples
- prefer evidence over authority
- keep discussion focused on the project


## Maintainer responsibilities

Maintainers may:

- request changes
- close discussions
- reject proposals
- remove disruptive content

These actions should support project health and collaboration.


## Scope

This Code of Conduct applies to:

- GitHub issues
- pull requests
- discussions
- documentation
- project communication


## Enforcement

If behavior becomes harmful to the project or contributors, maintainers may take appropriate action.

Actions can include:

- warnings
- comment removal
- temporary restrictions
- permanent removal from participation
